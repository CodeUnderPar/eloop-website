Devlog #003 – Refactoring and Structural Cleanup

February 28, 2026 | Office Cards

This session focused on code cleanup and structural refactoring rather than adding new gameplay features. What started as a small cleanup pass turned into a much larger organizational improvement.

Originally, I intended to remove unused variables and tidy up sections of the file. As I worked through it, I realized the project file had grown large enough that it was becoming difficult to navigate efficiently.

What Changed

The Card class and the EmployeeCard class were moved into their own separate files. This significantly improved readability and reduced scrolling friction when working inside the main project.

The next major candidate for separation is the Game class, though that will be handled in a future session.

I also removed unnecessary SFX-related variables and began identifying areas where variable usage can be reduced or consolidated going forward.

After refactoring, the project feels noticeably easier to manage. Adding new features now requires less mental overhead.

Design Thoughts

I’m considering adding a round timer that would force decisions within a set time limit. If a player fails to select a card before the timer expires, the selected card’s damage and health effects would drop to zero.

This could increase tempo and make combo chaining more skill-based under pressure. It’s still experimental and would need testing to determine if it enhances or frustrates gameplay.

Looking further ahead, I’m also thinking about implementing a simple progression system. Players might begin with the Intern card and unlock additional employees through consecutive wins. This is an early concept but could provide long-term structure.

Challenges

The most difficult design problem right now is making employee buffs interesting. At the moment, they primarily modify health values, but that feels too static.

The goal is to create unique and meaningful buffs without overcomplicating the system. Keeping the total number of employees limited (likely under eight) may help ensure each one feels distinct and purposeful.

Current State

The game remains in early systems development. Some text glitches persist, and overall balance is off since the combo system expansion.

The AI still needs to be updated to properly evaluate combo potential.

Structurally, however, the project is improving. It’s becoming easier to work with and easier to expand.

Steady progress forward.

Devlog #002 – Combos, Expansion, and Balance

February 23, 2026 | Office Cards

This session was focused on expanding and restructuring the combo system.

Originally, adding new combos required touching multiple areas of the code, which made experimentation slower than it needed to be. I refactored the system so new combinations can now be added more cleanly. That change alone makes future iteration much easier.

I introduced a few new combo variations, including Double and Triple bonuses, as well as a health bonus triggered when three cards start with the letter “P.” They’re functional, though not yet fully balanced.

I also added a subtle shake effect when combos activate. It’s a small detail, but it gives the game more physical feedback and makes successful plays feel more impactful.

Expanding the Employee System

I’ve started laying the groundwork for unique employee cards. Each employee will eventually have a signature card. For example, Intern Tim now has a special card called Tim’s Stapler. At the moment, it simply deals more damage than a standard card, but this opens the door to more interesting synergies in the future.

Right now, employee cards mainly provide health bonuses. I’d like to push them toward something more strategic — something that changes how you approach a round rather than just increasing a number.

A Design Question

One decision I’m still working through: should combos reset each round, or carry forward between rounds?

If they carry over, the game becomes momentum-driven and potentially snowballs. If they reset, each round becomes more contained and tactical.

The Current Problem

The game has become much easier to win. The issue isn’t that combos are broken — it’s that the AI doesn’t know how to use them.

Improving AI awareness of combo potential is the next real challenge.

Where Things Stand

The game is still in an early systems phase. Some UI text bugs remain. Visual assets are placeholders. The codebase still needs cleanup in areas.

But the structure is improving. The combo system is more flexible. The foundation is getting stronger.

Steady progress.
